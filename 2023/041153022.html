如何优雅的搭建个人博客

配置环境的安装

需要的配置环境：

  • node.js的安装与环境变量的配置

  • git安装

  • GitHub创建个人仓库

  • hexo

node.js的安装与环境变量的配置：

点击查看详细的教程

git的安装：

点击查看详细教程

git如何查看和登录自己的账号：

修改或者登录git账号
  • git config –global user.name “Stephenqhd30”
  • git config –global user.email “Stephenqhd@outlook.com“
查看自己的账号：
  • git config user.name
  • git config user.email

GitHub创建个人仓库:

创建一个和你用户名相同的仓库，后面加.github.io，只有这样，将来要部署到GitHub page的时候，才会被识别，也就是Stephenqhd30.github.io

hexo的安装和环境配置：

点击查看详细的教程

按照博客上的内容即可配置好环境

手动生成网页的根目录
  • hexo init [folder]

新建一个网站。如果没有设置 folder ，Hexo 默认在目前的文件夹建立网站。

  • cd [folder]

进入到刚刚创建好的文件夹里面，为网页的根目录

  • npm install

将网页部署到根目录中

手动生车工ssh密钥并将hexo部署到github上面
  • ssh-keygen -t rsa -C “Stephenqhd@outlook.com“

创建SSH,一路回车

这个时候它会告诉你已经生成了.ssh的文件夹。在你的电脑中找到这个文件夹。

  • C:\Users \ stephen qiu \ .ssh(默认的存放路径)

ssh，简单来讲，就是一个秘钥，其中，id_rsa是你这台电脑的私人秘钥，不能给别人看的，id_rsa.pub是公共秘钥，可以随便给别人看。把这个公钥放在GitHub上，这样当你链接GitHub自己的账户时，它就会根据公钥匹配你的私钥，当能够相互匹配时，才能够顺利的通过git上传你的文件到GitHub上。

而后在GitHub的setting中，找到SSH keys的设置选项，点击New SSH key
把你的id_rsa.pub里面的信息复制进去。

将hexo部署到GitHub

这一步，我们就可以将hexo和GitHub关联起来，也就是将hexo生成的文章部署到GitHub上，打开站点配置文件 _config.yml

## Deployment  部署设置
## Docs: https://hexo.io/docs/one-command-deployment
deploy:
  type: git
  repo: git@github.com:Stephenqhd30/Stephenqhd30.github.io.git
  branch: main
将文章上传的方法
  • hexo clean

清除缓存文件 (db.json) 和已生成的静态文件 (public)。

在某些情况（尤其是更换主题后），如果发现您对站点的更改无论如何也不生效，您可能需要运行该命令。

  • hexo g / hexo generate
  • hexo s / hexo server
  • hexo d / hexo deploy
