随机匹配

怎么随机匹配？（根据什么匹配）

答案：标签 tags

还可以根据 user_team 匹配加入相同队伍的用户

本质： 找到有相似标签的用户

举例：

用户A ：[Java, 大一, 男]

用户B ：[Java, 大二, 男]

用户C ： [Python, 大二, 女]

  1. 找到有共同标签最多的用户(TopN)
  2. 共同标签越多的相似多越高， 分数越高，越排到前面

1、使用什么算法完成这个事情

  1. 编辑距离算法

最小编辑距离：字符串 1 通过多少次增删改操作可以变成字符串 2

  1. 余弦相似度算法

带有权重的计算相似度，比如学什么方向最重要，性别相对次要

2、怎么对所有用户匹配，取Top N

直接取出来所有的用户，依次和当前的用户计算分数，取出Top N

优化方法

  1. 切忌不要在数据量大的额时候循环输出日志

  2. Map 存了所有的分数，占用内存

    解决： 维护一个固定长度的集合，按照分数降序，只保留分数最高的用户

e.g: [3, 4, 5, 6, 7]取 Top 5， id 为 1 的用户就不放进去了

  1. 细节：剔除自己
  2. 尽量只查需要的用户
    1. 过略掉标签为空的用户
    2. 根据部分标签取用户
    3. 只查询需要的数据（比如 id 和 tags）
  3. 提前查？
    1. 提前将所有的用户给缓存( 不适用于经常更新的数据)
    2. 提前运算出来结果，缓存

大数据推荐，比如说有几亿条数据，难道需要查出来所有的商品？

难道要对所有的数据计算一遍相似度？

检索 => 召回 => 粗排 => 精排 => 重排序等等

检索：尽可能多的查符合要求的数据（比如按记录查）

召回：查询可能要用的数据（不做运算）

粗排：粗略排序，简单地运算（运算相对轻量）

精排：精细排序，确定固定排位
